# My NLP Project: Tweet Hybrid Search & Retrieval 🚀

این پروژه یک سیستم پردازش زبان طبیعی (NLP) است که برای ذخیره‌سازی، نمایه‌سازی (Indexing) و جستجوی پیشرفته روی داده‌های متنی (توییت‌ها) طراحی شده است. این سیستم از جستجوی ترکیبی (Hybrid Search) با استفاده از بردارهای متراکم (Dense) و پراکنده (Sparse) و همچنین ابزارهای تشخیص شباهت متنی بهره می‌برد.

## 🌟 ویژگی‌های کلیدی
* **جستجوی ترکیبی (Hybrid Search):** استفاده از دیتابیس **Qdrant** برای جستجوی معنایی و کلیدواژه‌ای با ترکیب بردارهای Dense و Sparse و استفاده از روش RRF (Reciprocal Rank Fusion).
* **تولید امبدینگ (Embeddings):** استفاده از مدل قدرتمند `BGE-M3` (از طریق کتابخانه `FlagEmbedding`) برای تولید بردارهای متنی باکیفیت.
* **تشخیص توییت‌های تکراری / مشابه:** استفاده از **MinHash** (کتابخانه `datasketch`) و دیتابیس **Redis** برای یافتن تطابق‌های دقیق و شبه‌دقیق در زمان کوتاه.
* **پایپ‌لاین پیش‌پردازش:** پاک‌سازی متن، تبدیل تاریخ‌ها به Timestamp و تولید UUID برای هر توییت.

## 🛠️ پیش‌نیازها و نصب

برای اجرای این پروژه (مخصوصاً در محیط‌هایی مانند Google Colab)، ابتدا باید دیتابیس Redis و کتابخانه‌های پایتون مورد نیاز را نصب کنید.

### ۱. نصب وابستگی‌های پایتون:
```bash
pip install FlagEmbedding datasketch redis qdrant-client

### ۲. راه‌اندازی سرور Redis (برای اوبونتو / Colab):

## 🚀 نحوه استفاده

### ۱. تزریق داده‌ها به دیتابیس (Ingestion)
برای خواندن توییت‌ها، پیش‌پردازش، تولید امبدینگ و ذخیره آن‌ها در Qdrant و Redis، اسکریپت `populate_db.py` را اجرا کنید:
bash
python populate_db.py
*(نکته: مطمئن شوید که مسیر اجرای اسکریپت در روت پروژه (همان پوشه‌ای که فایل‌های پایتون قرار دارند) باشد. در صورت نیاز از `%cd` استفاده کنید.)*

### ۲. اجرای برنامه اصلی
پس از پر شدن دیتابیس‌ها، می‌توانید برنامه اصلی را برای انجام جستجو اجرا کنید:
bash
python main.py

## 🗂️ ساختار پروژه

* `populate_db.py`: اسکریپت اصلی برای پردازش داده‌ها (تولید UUID، پاک‌سازی، تولید امبدینگ با BGE-M3) و ذخیره در دیتابیس‌ها.
* `main.py`: اسکریپت اصلی اجرایی برای جستجو و تعامل با داده‌ها.
* `core/embedder.py`: ماژول مربوط به تولید امبدینگ‌ها با استفاده از `FlagEmbedding`.
* `core/preprocess.py`: توابع مربوط به پاک‌سازی و آماده‌سازی متن پیش از پردازش.
* `QdrantManager` (داخل کد): کلاس مدیریت اتصال به Qdrant، ساخت کالکشن‌ها و درج توییت‌ها (Payload و Vectors).
